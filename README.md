# car-cost-model

ノア Welcab の買い替え判断を、商談前後にブラウザだけで素早く比較するための静的Webアプリです。v1.1では、現在47歳から約20〜25年の車利用期間全体を見て、車検前買替年齢ごとの総支出を比較します。

## 使い方

1. `docs/index.html` をブラウザで直接開く。
2. 入力欄の査定額、ローン条件、維持費、修理期待値などを変更する。
3. Dashboard とシナリオ比較表が即時に再計算される。
4. 条件を持ち帰る場合は `JSONコピー` または `JSON保存` を使う。
5. 比較結果を表計算ソフトで見る場合は `CSV保存` を使う。

ローカルで直接開いた場合、ブラウザの制約で `docs/assumptions.json` を読み込めないことがあります。その場合も `docs/app.js` 内の同じ初期値で動作します。GitHub Pages では `docs/assumptions.json` を読み込みます。

## GitHub Pages 設定

GitHub のリポジトリ設定で Pages の公開元を次のように設定します。

- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/docs`

公開後は `https://abehiroshi.github.io/car-cost-model/` で開けます。

## v1 の考え方

v1.1 は厳密なローン償却表ではなく、一次判断用の簡易モデルです。最初の買替時期だけでなく、残り利用期間全体の車検前買替年齢を比較します。25年支出合計、25年平均月額、買替回数、車検回数、最終売却額、売却後実質コスト、売却後平均月額、25年末純資産を見ます。

年合計にはローン、維持費、修理期待値、車検、高額イベント費用を含めます。下取り額や最終車両価値は支出から控除せず、車両価値と純資産として別に表示します。下取り額は資産移動なので、年合計には混ぜません。

現在車ローンは現実の契約に合わせ、現契約の残価型ローンとして扱います。現在車両の月額、終了年、最終回はそのまま年次キャッシュフローへ反映します。

次回以降の買替ローンは、残価型ローンと通常ローンを分けて扱います。残価がある場合は残価型ローンで最長5年、残価0円の場合は通常ローンで8年固定です。通常ローンで8年以内に買い替える場合、前回ローン残債を現金一括返済せず、次回ローンへ繰り越します。新ローン総額は「新車価格 - 下取り額 + 前回ローン残債」で計算し、残債は簡易モデルとして「残月数 × 月額」で見ます。

旧条件の総返済額4,181,840円は参考入力として残しますが、通常ローンの買替時元本は各買替年の「新車価格 - 下取り額 + 前回ローン残債」で再計算します。標準では残価0円、最終月額0円、初回頭金10,000円の通常ローンとして扱います。初回支払いは「初回頭金 + 通常月額の初回分」として見ます。画面ではローン種別、初回頭金、月額、最終月額、残価、ローン年数を確認できます。通常ローンではローン年数を8年に固定し、新ローン総額に応じて月額が変わります。

年次キャッシュフローでは「ローンだけの月額」と、維持費・修理・車検・高額イベントを含む「その年の必要月額」を別指標として見ます。買替年のローン年額には初回頭金を含め、ローン最終年には最終月額と残価を含めます。購入・下取り・ローン・高額イベントキャッシュフローでは、新車購入費用、買替時点の下取り想定額、前回ローン残債、新ローン総額、現車ローン、買替ローン、合計ローン、高額イベント回避を同じ年次表で確認します。高額イベント費用は年合計に1回だけ含め、この専用表では再掲として表示します。

自動車税は障害者減免を前提に0円として扱います。

高額イベント費用は `maintenance-events.json` で車検・通常整備とは別に管理します。タイヤ、補機バッテリー、エアコン、足回り、HVバッテリー、Welcab大型修理などを車齢ベースで発生させ、年合計へイベント費用として加算します。ただし買替予定が近い場合は実施せずに買替で回避する想定とし、買替年・買替前年の高額イベント費用は0円にします。

利用期間の最終年には、その時点の車両価値を最終売却額として別指標にします。今日の意思決定では、短期月額だけでなく25年支出合計と売却後実質コストを重視します。

5年目車検前買替は車検・修理は少ない一方で、買替回数とローン期間が多くなります。9〜15年目車検前買替は車検・修理は増えますが、買替回数が少なくなります。

比較対象は `5年目車検前`、`7年目車検前`、`9年目車検前`、`11年目車検前`、`13年目車検前`、`15年目車検前` です。表示上の年数は車検前買替年齢であり、実際の買替はその車検が発生する前年に前倒しされます。このため、短期買替では車検回数と高額イベントが少なくなる一方、買替回数とローン期間は増えます。

主な前提値は `docs/assumptions.json` と `inputs/assumptions.json` に集約しています。高額イベントは `docs/maintenance-events.json` と `inputs/maintenance-events.json` に置きます。各ファイルの `docs/` と `inputs/` は同じ内容です。車齢別の査定額は `tradeInTable` としてJSONに置いています。

画面の「前提・単価・計算ルール」では、開始年、ローン条件、維持費、修理期待値、高額イベント、査定額テーブル、資産価値、年合計と月額の式を確認できます。商談中に、どの金額が年合計に入り、どの金額が資産価値や最終売却額として別表示されるのかを画面だけで確認するための補助です。新車購入費用と下取り想定額は支出合計とは別に表示し、ローン支払いだけを家計支出として年合計に入れます。

## 根拠メモ

`knowledge/` は、ChatGPTに「Web検索せず、リポジトリ内の根拠だけで説明して」と依頼するための根拠メモです。最新Web情報ではなく、このモデル内で採用している前提、仮置き、未確認事項を説明するために使います。

数値を変える場合は `inputs/assumptions.json` と `docs/assumptions.json` を更新し、根拠を `knowledge/` に追記します。
